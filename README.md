# KKTIX 搶票系統

自動化 KKTIX 演唱會、活動搶票工具。使用 Puppeteer 控制真實瀏覽器執行登入、偵測票券狀態、填寫表單、完成付款，搭配 Vue 前端介面進行設定與即時監控。

---

## 功能總覽

| 功能 | 說明 |
| --- | --- |
| 自動登入 | 自動前往 KKTIX 登入頁並輸入帳密 |
| 高頻刷票 | 依設定間隔（最快 500ms）持續重整活動頁偵測開賣 |
| 自動搶購 | 票券開放後立即選票、填表、送出 |
| 自動填表 | 自動填入姓名、手機、身分證、Email |
| 自動付款 | 自動填入信用卡資料完成付款（支援 TapPay iframe） |
| 驗證碼提示 | 偵測到 reCAPTCHA 時暫停並通知使用者手動完成 |
| 多帳號輪替 | 支援主帳號＋備用帳號，每 3 次嘗試自動切換 |
| 排程搶票 | 預設多筆活動，倒數到開賣時間自動觸發 |
| 即時監控 | WebSocket 即時串流執行日誌、狀態、嘗試次數 |
| 反偵測 | 使用 Puppeteer Stealth Plugin 隱藏自動化特徵 |
| **活動瀏覽** | **從 KKTIX 探索頁自動抓取活動列表，卡片橫式排列，含搜尋與直接選取** |
| **設定下拉選單** | **設定頁可從下拉選單選擇活動，自動帶入活動網址，日期以金黃色標籤標示** |
| **資料持久化** | **設定、帳號、排程以 localStorage 保存，重整後不需重填** |
| **Cloudflare 繞過** | **使用系統 Chrome 執行、反偵測腳本注入、登入流程強化（含 422 修復）** |
| **RWD 響應式設計** | **支援手機、平板、桌機，手機版漢堡選單、單欄排版，可透過區網遠端操控** |
| **排程活動下拉選單** | **排程頁新增活動選單，選擇後自動帶入名稱、網址、日期** |
| **自訂問答處理** | **自動偵測特殊驗證問題（格式「請輸入X」自動填入），無法處理時暫停等手動填寫** |
| **導航速度優化** | **移除固定 2 秒等待，Cloudflare 改為單次快速檢查，每輪省下 2–10 秒** |

## 系統架構

```text
grabber/
├── server/              # Node.js 後端
│   ├── index.js         # Express + WebSocket 伺服器（port 3000）
│   ├── grabber.js       # 搶票主控邏輯（含反偵測、Cloudflare 處理）
│   ├── kktix.js         # KKTIX 頁面操作（selector、登入、表單、付款）
│   └── events.js        # KKTIX 活動爬蟲（含 5 分鐘快取）
└── src/                 # Vue 3 前端
    ├── views/
    │   ├── EventsView.vue      # 活動瀏覽頁（新增）
    │   ├── SettingsView.vue    # 設定頁（含活動下拉選單）
    │   ├── AccountsView.vue    # 帳號管理頁
    │   ├── MonitorView.vue     # 監控頁
    │   └── ScheduleView.vue    # 排程頁
    ├── store/
    │   └── grabberStore.js     # 全域狀態（Vue reactive + localStorage 持久化）
    └── services/
        └── socket.js           # WebSocket + REST API 封裝
```

**前端** → REST API + WebSocket → **後端** → Puppeteer → **KKTIX**

## 安裝

### 環境需求

- Node.js 16+
- npm 7+

### 步驟

```bash
# 1. 安裝前端依賴
npm install

# 2. 安裝後端依賴
cd server
npm install
cd ..
```

## 啟動方式

需要同時啟動**後端**與**前端**兩個服務。

### 啟動後端（必須先啟動）

```bash
cd server
npm start
```

後端啟動於 `http://localhost:3000`，同時開啟 WebSocket 監聽。

### 啟動前端

```bash
# 開發模式（支援熱更新）
npm run serve

# 或編譯為生產版本
npm run build
```

前端預設於 `http://localhost:8080`。

### 手機遠端操控（區網）

後端與前端啟動後，在手機瀏覽器開啟：

```text
http://電腦區網IP:8080
```

> 電腦區網 IP 查詢：Windows 執行 `ipconfig`，找 IPv4 位址（通常為 `192.168.x.x`）。
>
> 手機需與電腦連接**同一個 WiFi**。搶票流程仍在電腦上執行，手機作為遠端設定與監控介面。

## 使用說明

### 步驟零：活動瀏覽（可選）

前往「**活動瀏覽**」頁面，可直接從 KKTIX 上瀏覽所有目前開放的活動：

1. 點擊「**立即載入**」，系統以 headless 瀏覽器抓取 KKTIX 探索頁（首次約 15–20 秒，後續 5 分鐘快取）
2. 活動以卡片方式橫式排列，顯示封面圖、名稱、日期
3. 在搜尋欄輸入關鍵字可即時過濾活動名稱
4. 點擊「**選擇此活動**」，系統自動將網址填入「設定」頁並跳轉

> 過期活動（今年以前）與已結束活動不會顯示。

---

### 步驟一：設定帳號

前往「**帳號管理**」頁面：

1. 點擊「**+ 新增帳號**」
2. 填入帳號名稱、KKTIX Email、密碼
3. 點擊「**設為主要**」將該帳號設為主帳號
4. 透過右側開關啟用/停用帳號

**帳號輪替策略**（可選）：

| 策略 | 說明 |
| --- | --- |
| 順序輪替 | 依序嘗試每個帳號（預設） |
| 隨機輪替 | 隨機選擇可用帳號 |
| 最快優先 | 優先使用成功率最高的帳號 |
| 同步並發 | 所有帳號同時嘗試搶票 |

> 開啟「多帳號輪替」時，每嘗試 3 次後自動切換至下一個備用帳號。

---

### 步驟二：填寫搶票設定

前往「**設定**」頁面：

#### 活動資訊

| 欄位 | 說明 | 範例 |
| --- | --- | --- |
| 活動網址 | KKTIX 活動頁完整網址（必填） | `https://kktix.com/events/xxx` |
| 票種名稱 | 要搶的票種關鍵字，留白則自動選第一種 | `一般票`、`早鳥票` |
| 購買數量 | 1–4 張 | `2` |
| 刷新間隔 | 每次重整的間隔時間 | 500ms（極速）～ 5000ms（保守） |

#### 進階選項

| 選項 | 說明 |
| --- | --- |
| 自動刷新 | 依設定間隔持續重整直到票券開放 |
| 多帳號輪替 | 啟用備用帳號自動切換 |
| 驗證碼提示 | 偵測到驗證碼時發出視覺提示 |
| 排程自動開搶 | 在此頁設定開搶時間，到時自動啟動 |

#### 報名表單資料（搶到票後自動填入）

| 欄位 | 說明 |
| --- | --- |
| 真實姓名 | 報名用真實姓名 |
| 手機號碼 | 聯絡電話 |
| 身分證字號 | 部分活動必填 |
| 聯絡 Email | 若表單有 Email 欄位則自動填入 |

#### 付款資訊（選填，留白則在瀏覽器手動付款）

| 欄位 | 說明 |
| --- | --- |
| 持卡人姓名 | 信用卡上的英文姓名 |
| 信用卡號 | 16 位數，自動格式化為 `xxxx xxxx xxxx xxxx` |
| 有效期限 | `MM/YY` 格式 |
| CVV / 安全碼 | 3–4 位數安全碼 |

> **資料安全**：信用卡資料僅存於本機記憶體，不會上傳或儲存至任何伺服器。

填寫完成後點擊「**儲存設定**」，再點「**前往搶票**」。

---

### 步驟三：開始搶票

前往「**監控**」頁面：

1. 確認頂部顯示「後端伺服器已連線」（綠色）
2. 確認搶票目標資訊正確
3. 點擊「**🚀 開始搶票**」

系統會自動執行以下流程：

```text
登入 KKTIX → 前往活動頁 → 偵測票券狀態
    ↓ 尚未開賣
等待 N 秒後重試（循環）
    ↓ 票券開放
選票 + 設定數量 → 填寫報名表單 → 填寫付款資料 → 完成
```

#### 監控頁顯示資訊

| 資訊 | 說明 |
| --- | --- |
| 嘗試次數 | 目前已刷新幾次 |
| 執行時間 | 從啟動起算的時間 |
| 次/分鐘 | 當前刷票頻率 |
| 啟用帳號 | 目前啟用的帳號數量 |

#### 日誌顏色說明

| 顏色 | 類型 | 說明 |
| --- | --- | --- |
| 藍色 INFO | 一般訊息 | 正常執行步驟 |
| 黃色 WARN | 警告 | 尚未開賣、等待中 |
| 綠色 OK | 成功 | 票券可用、搶票成功 |
| 紅色 ERR | 錯誤 | 登入失敗、票已售罄 |

---

### 處理驗證碼

系統偵測到驗證碼（reCAPTCHA）時會：

1. 暫停搶票流程
2. 在監控頁顯示黃色警告橫幅
3. 最多等待 **120 秒**

使用者需要：

1. 切換到已開啟的瀏覽器視窗
2. 手動完成驗證碼
3. 回到監控頁點擊「**✅ 我已完成驗證**」

完成後系統自動繼續搶票。

---

### 步驟四：排程搶票（可選）

前往「**排程**」頁面，可預先登記多筆活動：

1. 點擊「**展開**」開啟新增表單
2. 填入：
   - 活動名稱（自訂識別用）
   - 開賣時間（選擇日期時間）
   - 活動網址
   - 票種名稱（留白自動選第一種）
   - 購買數量
3. 點擊「**加入排程清單**」

> 帳號、付款資訊、刷新間隔等設定沿用「設定」頁的內容。

排程卡片狀態：

| 狀態 | 說明 |
| --- | --- |
| 等待中 | 顯示倒數計時，到時自動觸發 |
| 已暫停 | 手動暫停，不會自動觸發 |
| 搶票中 | 正在執行，對應監控頁即時顯示 |
| 成功 | 搶票完成 |
| 失敗 | 執行失敗，可點「重設」再試 |

> 距開賣不足 1 分鐘時倒數計時變為橘色閃爍提示。

也可以不等排程，直接點「**立即搶**」手動觸發任一筆排程。

---

## 注意事項

- 搶票過程中瀏覽器視窗會自動開啟，**請勿手動關閉**
- 搶票成功後瀏覽器會延遲 5 秒關閉，讓使用者確認結果
- 若付款資訊未填，系統在完成報名表單後會停在付款頁等待手動操作
- KKTIX 頁面結構若有更新，selector 可能需要調整（位於 `server/kktix.js`）
- 建議刷新間隔設定 1000ms 以上，避免對網站造成過大負擔
